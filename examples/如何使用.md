## 🚀 方法一：最简单 - 使用 Visual Studio（推荐）

1. **打开解决方案**
   ```
   双击打开: d:\TraeProject\RinzePlayer\build\RinzePlayer.sln
   ```

2. **添加项目**
   - 在 Visual Studio 右侧的"解决方案资源管理器"中
   - 右键点击 **解决方案 RinzePlayer**
   - 选择 **添加 -> 现有项目**
   - 浏览并选择: `d:\TraeProject\RinzePlayer\examples\feature_check.vcxproj`

3. **配置项目属性（如果需要）**
   - 右键点击 feature_check 项目 - 属性
   - 确保:
     - 配置: Release / Debug
     - 平台: x64
     - C/C++ -^> 常规 -^> 附加包含目录:
       ```
       $(ProjectDir)..;D:\FFMPEGLIB\include
       ```
     - 链接器 -^> 常规 -^> 附加库目录:
       ```
       D:\FFMPEGLIB\lib
       ```
     - 链接器 -^> 输入 -^> 附加依赖项:
       ```
       avfilter.lib;avutil.lib
       ```

4. **编译并运行**
   - 右键点击 feature_check 项目 -^> **设为启动项目**
   - 按 **F5** 或 Ctrl+F5 运行

## 🚀 方法二：使用自动编译脚本

1. **找到 Developer Command Prompt**
   - 在 Windows 开始菜单搜索: "x64 Native Tools Command Prompt for VS 2022"
   - 右键点击 -^> 以管理员身份运行

2. **进入 examples 目录并编译**
   ```cmd
   cd /d d:\TraeProject\RinzePlayer\examples
   build_simple.bat
   ```

3. **运行程序**
   - 脚本会自动编译并运行程序
   - 或者你也可以在 `examples\bin\` 目录找到可执行文件

---

## 🚀 方法三：直接使用已有 build 环境（最方便）

由于项目已经有 build 目录，我们可以直接修改并编译：

1. **打开 build 目录下的解决方案**
   ```
   d:\TraeProject\RinzePlayer\build\RinzePlayer.sln
   ```

2. **添加 feature_check 项目**
   - 右键点击解决方案 -> 添加 -> 现有项目
   - 选择 `d:\TraeProject\RinzePlayer\examples\feature_check.vcxproj`

3. **修改项目的 FFmpeg 路径（如果需要）**
   - FFmpeg 路径是: `D:\FFMPEGLIB`
   - 项目属性中的路径已经设置为这个

4. **编译运行**
   - 右键点击 feature_check -> 生成
   - 右键点击 feature_check -> 设为启动项目 -> F5

## 📄 需求说明（再次回顾）

### ✅ 需求 A: setupFilter with Vulkan
实现了以下功能：
- `av_hwdevice_ctx_create` 创建 Vulkan 设备
- 将 Vulkan 设备关联到 `AVFilterGraph`
- 在 filter 链中添加 `hwupload`
- 保留原来的 libplacebo 所有参数
- 完整的错误处理

### ✅ 需求 B: FFmpeg 功能检查
- 检查 `av_hwdevice_ctx_create` 是否可用
- 检查 `libplacebo` filter 是否可用
- 检查 `hwupload` filter 是否可用

### ✅ 需求 C: Vulkan 运行时检查
- 检查 vulkan-1.dll 是否存在
- 不依赖 Qt
- 完整的检查流程

## 💡 问题排查

### 找不到 cl.exe
请使用 **Visual Studio 2022 Developer Command Prompt (x64)** 运行

### 找不到 FFmpeg 或链接错误
- 确认 FFmpeg 在 `D:\FFMPEGLIB`
- 或者修改 `build_simple.bat` 中的路径
- 或者在 Visual Studio 项目属性中修改

### FFmpeg DLL 错误
确保 FFmpeg 的 DLL 在同一目录，或者在 PATH 中
- build 目录中的 Release/Debug 文件夹已经有正确的 DLL
